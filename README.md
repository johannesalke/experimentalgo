# What's all this then?

Oh, nothing much. I just decided that I needed a place where I could do small experiments while trying to figure out aspects of the Go language and various modules. 
One of the things I appreciated about Python is that you could just try things out and quickly check whether something would work as you expected or not in the python interpreter shell. Go doesn't have that, so I'm using this as a replacement. 

##### Structure

The experiments are mostly spread across various files, contained in functions. Those functions are called in the main.go file to check the results. This setup keeps the 'core' file simple and lets me control what is and isn't executed each time without having to do much cleanup.
